## 后端接口测试

以下示例使用 `curl`，Windows PowerShell 可替换为 `Invoke-RestMethod`。

### 登录并获取 Token

```bash
curl -X POST http://8.218.156.55:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d "{\"account\":\"parent@huadee.test\",\"password\":\"123456\"}"
```

预期：返回 `code = 0`，`data.token` 非空。

### 查询个人信息

```bash
curl http://8.218.156.55:8080/api/users/profile \
  -H "Authorization: Bearer <TOKEN>"
```

预期：返回当前用户资料。

### 更新个人信息

```bash
curl -X PUT http://8.218.156.55:8080/api/users/profile \
  -H "Authorization: Bearer <TOKEN>" \
  -H "Content-Type: application/json" \
  -d "{\"name\":\"家校通测试用户\",\"phone\":\"13800009999\",\"email\":\"profile-test@huadee.test\"}"
```

预期：返回更新后的 `name`、`phone`、`email`，重新查询仍保持一致。

### 核心列表接口

| 接口 | 预期 |
| --- | --- |
| `GET /api/notices` | 返回分页对象，包含 `list` 和 `records` |
| `GET /api/homework` | 返回作业分页列表 |
| `GET /api/conversations` | 登录后返回会话数组 |
| `GET /api/users/contacts` | 登录后返回联系人数组 |
| `POST /api/users/contacts` | 通过手机号、邮箱或用户ID添加联系人 |
| `GET /api/attendance/records?studentId=<ID>` | 返回考勤数组 |
| `GET /api/grades/reports?studentId=<ID>` | 返回成绩报告数组 |
| `GET /api/health/records?studentId=<ID>` | 返回健康档案数组 |
| `GET /api/evaluations?studentId=<ID>` | 返回评价数组 |
| `GET /api/activities` | 返回活动分页对象 |

## 回归重点

- 所有功能页进入、返回、刷新时不能闪退。
- 后端返回 `data: null` 或空数组时，前端必须展示空状态。
- 个人信息保存后，需要同时更新数据库、当前页面和“我的”页展示。
- 主工作台和底部导航必须使用图标组件，不再使用单个蓝色汉字代替图标。
- 修改资源、图标、桌面名称后，需要卸载旧 App 并重新安装验证。
